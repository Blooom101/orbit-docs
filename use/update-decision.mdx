---
title: "How to update a decision"
description: "File, edit, or close a standing instruction your agents follow."
---

Decisions are standing instructions every relevant agent reads before each run. Here's how to file a new one, change an existing one, or close it out.

## File a new decision

1. Open **Decisions** from your dashboard.
2. Click **New Decision**.
3. Write the directive as a clear, imperative one-liner: *"Always quote the $49 plan for outbound."*
4. Set the **scope** — which agents it applies to (a single agent, a project, a surface, or everyone).
5. Optionally add a **rationale** so agents and teammates understand the why.
6. Save. It takes effect on the next agent run.

## Edit a decision

Open the decision and update its text or scope. The change applies to the next run — no agent redeploy needed. Every connected agent reads the updated version automatically.

## Close a decision

When a decision no longer applies, open it and mark it **Closed**. Agents stop reading it immediately. Closed decisions stay in your history for the audit trail.

## Good decisions vs. vague ones

| Good | Vague |
|---|---|
| "Never contact leads from competitor domains" | "Be careful with leads" |
| "Use JWT for auth — don't suggest alternatives" | "Follow our auth approach" |
| "Focus this week on healthcare prospects" | "Prioritize good prospects" |

Specific, testable directives produce reliable agent behavior. Vague ones get interpreted inconsistently.
