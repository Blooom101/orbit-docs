# File Upload Tools

Orbit supports uploading binary files (PDFs, images, Excel) and text files (markdown, CSV, JSON).

## Two-Step Binary Upload

For files > 1MB or binary formats, use the signed URL flow:

### Step 1: orbit_file_init

```javascript
const file = await orbit_file_init({
  agent: { id: "my-agent", project: "my-project" },
  name: "q3-report.pdf",
  mime: "application/pdf",
  size: 1048576,
  ref: "output/q3-report"  // Stable slug for versioning
});
```

Parameters:
- `name` - Filename with extension (e.g., "report.pdf")
- `mime` - MIME type (e.g., "application/pdf")
- `size` - File size in bytes
- `ref` - Optional: stable slug for versioning. Reuse to update file in place.

Response:
```json
{
  "ok": true,
  "file_id": "f-123",
  "mode": "signed",
  "upload_url": "https://signed-url-for-upload"
}
```

### Step 2: Upload to URL

```javascript
const response = await fetch(file.upload_url, {
  method: "PUT",
  headers: {
    "Content-Type": "application/pdf",
    "x-upsert": true  // Required to update existing file
  },
  body: pdfBytes  // Raw bytes, not base64
});
```

### Step 3: orbit_file_commit

```javascript
await orbit_file_commit({
  agent: { id: "my-agent", project: "my-project" },
  file_id: file.file_id
});
```

### Step 4: Attach to Result

```javascript
await orbit_result({
  agent: { id: "my-agent", project: "my-project" },
  title: "Published Q3 report",
  content: "See attached PDF for full analysis.",
  file_ids: [file.file_id]
});
```

## One-Step Text Upload

For text, markdown, CSV, JSON (< 2MB), use the simpler flow:

```javascript
const file = await orbit_file_put_text({
  agent: { id: "my-agent", project: "my-project" },
  name: "prospects.csv",
  content: "Name,Email,Status\nAlice,alice@...,Contacted\nBob,bob@...,Qualified",
  ref: "output/q3-prospects"
});
```

Returns:
```json
{
  "ok": true,
  "file_id": "f-456"
}
```

No upload URL, no commit step. Lands ready immediately.

Attach to result:
```javascript
await orbit_result({
  agent: { id: "my-agent", project: "my-project" },
  title: "Exported Q3 prospects",
  content: "See attached CSV.",
  file_ids: [file.file_id]
});
```

## File Versioning

Use the `ref` parameter to version files:

```javascript
// First upload
await orbit_file_init({
  name: "report.pdf",
  size: 1000000,
  ref: "output/q3-report"
});

// Later, update the same file
await orbit_file_init({
  name: "report.pdf",
  size: 1100000,
  ref: "output/q3-report"  // Same ref!
});
```

Orbit versions the file in place (v1, v2, v3...). The Vault shows the latest version; old versions are archived.

**Without `ref`**, each upload becomes a separate file. Use `ref` for anything you might update later (reports, decks, exports).

## File Types

### Binary (Use orbit_file_init + signed URL)
- PDF
- Images (PNG, JPG, SVG)
- Excel (.xlsx, .xls)
- PowerPoint (.pptx)
- Word (.docx)
- ZIP archives
- Audio/video
- Any non-text

### Text (Use orbit_file_put_text)
- Markdown (.md)
- Plain text (.txt)
- CSV
- JSON
- YAML
- HTML
- XML

## Patterns

### Upload a Generated PDF

```javascript
const PDFDocument = require("pdfkit");

// Generate PDF
const doc = new PDFDocument();
doc.fontSize(16).text("Q3 Report");
doc.fontSize(12).text("Total revenue: $1.2M");

const buffer = await new Promise((resolve) => {
  const chunks = [];
  doc.on("data", (chunk) => chunks.push(chunk));
  doc.on("end", () => resolve(Buffer.concat(chunks)));
  doc.end();
});

// Upload
const file = await orbit_file_init({
  agent: { id: "analyst", project: "my-project" },
  name: "q3-report.pdf",
  mime: "application/pdf",
  size: buffer.length,
  ref: "output/q3-report"
});

await fetch(file.upload_url, {
  method: "PUT",
  headers: { "Content-Type": "application/pdf", "x-upsert": true },
  body: buffer
});

await orbit_file_commit({
  agent: { id: "analyst", project: "my-project" },
  file_id: file.file_id
});

await orbit_result({
  agent: { id: "analyst", project: "my-project" },
  title: "Published Q3 report",
  file_ids: [file.file_id]
});
```

### Export CSV from Data

```javascript
const data = [...];
const csv = [
  "Name,Email,Status",
  ...data.map(d => `${d.name},${d.email},${d.status}`)
].join("\n");

const file = await orbit_file_put_text({
  agent: { id: "exporter", project: "my-project" },
  name: "data-export.csv",
  content: csv,
  ref: "output/data-export"
});

await orbit_result({
  agent: { id: "exporter", project: "my-project" },
  title: "Exported data",
  file_ids: [file.file_id]
});
```

### Update Existing File

```javascript
// Second version of the same file
const file = await orbit_file_init({
  agent: { id: "analyst", project: "my-project" },
  name: "q3-report.pdf",
  mime: "application/pdf",
  size: newBuffer.length,
  ref: "output/q3-report"  // Same ref as before
});

// Upload new bytes
await fetch(file.upload_url, {
  method: "PUT",
  headers: { "Content-Type": "application/pdf", "x-upsert": true },
  body: newBuffer
});

// Commit
await orbit_file_commit({
  agent: { id: "analyst", project: "my-project" },
  file_id: file.file_id
});

// The Vault now shows v2, with v1 archived
```

## Best Practices

1. **Use `ref` for persistent files** - Reports, exports, templates that you update regularly
2. **Omit `ref` for one-offs** - Per-run screenshots, temp exports
3. **Put full content in `orbit_result`** - Don't rely on attachments; the document lives in the result content
4. **Text-first approach** - Use `orbit_file_put_text` for CSV, JSON, markdown (simpler, faster)
5. **Binary for polish** - Use PDFs when you want formatting and layout

## Limits

- Max file size: 50MB
- Max text via `orbit_file_put_text`: 2MB
- Upload timeout: 5 minutes
- No concurrent uploads of the same file
