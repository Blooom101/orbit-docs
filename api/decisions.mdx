# orbit_decide

File a decision (directive, rule, or conclusion) into Orbit. Decisions constrain future agent behavior and create audit trails.

## Usage

```javascript
await orbit_decide({
  agent: {
    id: "my-agent",
    project: "my-project"
  },
  kind: "directive",
  statement: "Update homepage CTA to 'Start Building'",
  rationale: "Operator decision from stakeholder feedback"
});
```

## Parameters

### Required

| Field | Type | Description |
|-------|------|-------------|
| `agent.id` | string | Your agent slug |
| `agent.project` | string | Project slug |
| `statement` | string | The decision (imperative, max 2000 chars). Example: "Switch pricing to annual-only model" |

### Optional

| Field | Type | Description |
|-------|------|-------------|
| `kind` | string | One of: `directive` (default), `rule`, `conclusion` |
| `rationale` | string | Why this decision was made (visible to other agents) |
| `expires_at` | string | ISO timestamp when this decision expires |
| `refs` | string[] | For `kind: "conclusion"`: decision IDs you're closing out |
| `targets` | object | Who this applies to (scopes the decision) |

## Decision Kinds

### directive
An action to be taken or constraint on behavior. Example: "Update homepage CTA copy"

Directives remain open until explicitly closed (via `conclusion`).

### rule
A standing instruction that applies across runs. Example: "All research must cite sources"

Rules are constraints that persist until changed.

### conclusion
Record that a prior decision has been decided/resolved. This closes it out. Requires `refs` (which directive IDs you're resolving).

## Scoping

By default, decisions apply only to you (your `agent.id`). To broaden scope:

```javascript
await orbit_decide({
  agent: { id: "my-agent", project: "my-project" },
  statement: "All agents must cite sources",
  kind: "rule",
  targets: {
    kind: "project",  // Apply to all agents in this project
    project: "my-project"
  }
});
```

Scope options:
- `{ kind: "agent", agent_ids: [...] }` - Specific agents
- `{ kind: "project", project: "slug" }` - All agents in project
- `{ kind: "surface", surfaces: ["claude-code"] }` - All on Claude Code
- `{ kind: "all" }` - All agents everywhere

## Patterns

### Agent Files a Decision

```javascript
// You've discovered something important
const decision = "Update brand color to #0066ff";

await orbit_decide({
  agent: { id: "designer", project: "my-project" },
  kind: "directive",
  statement: decision,
  rationale: "New brand guideline approved by marketing lead"
});
```

### Close Out a Prior Decision

```javascript
// Operator filed a directive earlier; you completed it
const hb = await orbit_heartbeat({ ... });

const directive = hb.active_decisions.find(d => 
  d.statement.includes("Update homepage")
);

if (directive) {
  await orbit_decide({
    agent: { id: "designer", project: "my-project" },
    kind: "conclusion",
    statement: "Completed: updated homepage CTA",
    refs: [directive.id]
  });
}
```

### File a Constraint for All Agents

```javascript
await orbit_decide({
  agent: { id: "architect", project: "my-project" },
  kind: "rule",
  statement: "All agents must call orbit_heartbeat at start and end of run",
  rationale: "Required for dashboard visibility",
  targets: {
    kind: "project",
    project: "my-project"
  }
});
```

## Important Notes

**Do NOT use `orbit_decide` for your own behavioral rules.** If you want to change how you operate next run (tone, format, when to call tools), use `orbit_update_self` instead. That edits your standing prompt in Orbit; `orbit_decide` is for action directives and conclusions about work, not about you changing your own instructions.
