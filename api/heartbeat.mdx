# orbit_heartbeat

The heartbeat tool is mandatory for every agent run. Call it at the start and end to signal status to the operator and receive directives.

## Usage

### Start of Run

```javascript
const response = await orbit_heartbeat({
  agent: {
    id: "daily-digest",
    name: "Daily Digest Agent",
    type: "code",
    project: "my-business"
  },
  status: "running",
  context: "Compiling daily news digest for Q3"
});
```

### End of Run

```javascript
await orbit_heartbeat({
  agent: {
    id: "daily-digest",
    project: "my-business"
  },
  status: "idle",
  summary: "Compiled 47 stories, published digest"
});
```

## Parameters

### Required

| Field | Type | Description |
|-------|------|-------------|
| `agent.id` | string | Stable kebab-case slug for this agent (e.g., `daily-digest-claude`) |
| `agent.project` | string | Project slug (e.g., `my-business`) |
| `status` | string | One of: `running` (start), `idle` (end), `error` (failed) |

### Optional (Start)

| Field | Type | Description |
|-------|------|-------------|
| `agent.name` | string | Display name on dashboard (Title Case). Defaults to id. |
| `agent.type` | string | One of: `code`, `cowork`, `scheduled`, `custom` |
| `agent.provider` | string | Your AI provider: `anthropic`, `openai`, `google`, etc. |
| `agent.surface` | string | Where you're running: `claude-code`, `cowork`, `api`, `custom` |
| `agent.session_id` | string | External session ID for tracking |
| `agent.model_family` | string | Model you're using (e.g., `claude-haiku-4-5`) |
| `context` | string | One-line description of what you're doing (max 2000 chars) |

### Optional (End)

| Field | Type | Description |
|-------|------|-------------|
| `summary` | string | One-line summary of the run (max 2000 chars). Required when `status: "idle"` |

## Key Fields

### pending_messages

Messages sent to you by the operator while you were offline. You MUST read and act on these before proceeding. Respond via `orbit_message`.

### active_knowledge

Authoritative facts and conclusions relevant to your run. Ranked by:
1. Pinned items first
2. Semantic relevance to your `context`
3. Recency

Use this as ground truth. If you would contradict an authoritative item, call `orbit_context` to check if there's a decision blocking it.

### active_decisions

Directives and rules scoped to you (or broader). These constrain your behavior:

- **Directive** - Action you should take or constraint on what you do
- **Rule** - Standing instruction that applies across runs
- **Conclusion** - A question that's been decided; you can rely on it

If your planned output contradicts a decision, Orbit will flag it when you call `orbit_result`. You can proceed anyway (and the operator can override), or adjust your work.

### known_projects

All project slugs currently in use. Use these exact slugs for `agent.project` instead of inventing new ones.

### your_trackers

Tables you own (tables you've written to via `orbit_table_write`). Useful for remembering what persistent data you maintain. Keys: `table_ref`, `title`, `columns`, `row_count`.

## Patterns

### Handling Pending Messages

```javascript
const hb = await orbit_heartbeat({
  agent: { id: "my-agent", type: "code", project: "my-project" },
  status: "running",
  context: "Daily digest"
});

for (const msg of hb.pending_messages) {
  console.log(`Operator says: ${msg.text}`);
  
  // Do what they asked
  // Then reply
  await orbit_message({
    agent: { id: "my-agent", project: "my-project" },
    content: "Done: updated the copy as requested"
  });
}
```

### Checking Active Decisions Before Acting

```javascript
const hb = await orbit_heartbeat({ ... });

// Plan my output
const output = generateReport();

// Check: would this violate any decisions?
for (const decision of hb.active_decisions) {
  if (decision.statement.includes("must cite sources")) {
    if (!output.sources) {
      // Oops, I need sources. Add them.
      output.sources = [...];
    }
  }
}
```

### Using Ground Truth Knowledge

```javascript
const hb = await orbit_heartbeat({ ... });

// Find the latest product messaging
const messaging = hb.active_knowledge.find(k => 
  k.title.includes("messaging") && k.pinned
);

if (messaging) {
  const content = await orbit_get_result({ ... });
  // Use messaging.content as ground truth
}
```

## Error Handling

```javascript
try {
  const hb = await orbit_heartbeat({
    agent: { id: "my-agent", project: "my-project" },
    status: "running",
    context: "Daily task"
  });
} catch (err) {
  console.error("Failed to heartbeat:", err.message);
  // If heartbeat fails, you're disconnected from Orbit
  // You can continue offline, but your work won't be visible
  // Consider failing the run
  process.exit(1);
}
```

## Best Practices

1. **Call heartbeat immediately** - Don't wait to call it until you're done. Call it first, then process the response.
2. **Always provide context** - Make it specific: "Analyzing Q3 sales" not "Running"
3. **Use real agent names** - Display names help the operator track multiple instances of the same agent type
4. **Read messages first** - Process `pending_messages` before deciding your action
5. **Check decisions** - Before committing to an output, verify it won't contradict an active decision
6. **End every run** - Even if it failed, call `status: "error"` so the operator knows

## Migration: From Manual Runs to Orbit

If you have an existing agent, migrate step by step:

```javascript
// Old: Just run and hope
async function oldAgent() {
  const data = await fetchData();
  const result = processData(data);
  return result;
}

// Step 1: Add heartbeat, but still return normally
async function newAgent() {
  const hb = await orbit_heartbeat({
    agent: { id: "my-agent", type: "code", project: "my-project" },
    status: "running",
    context: "Processing data"
  });
  
  const data = await fetchData();
  const result = processData(data);
  
  await orbit_heartbeat({
    agent: { id: "my-agent", project: "my-project" },
    status: "idle",
    summary: "Processed data"
  });
  
  return result;
}

// Step 2: Publish result to Vault
async function orbitalAgent() {
  const hb = await orbit_heartbeat({
    agent: { id: "my-agent", type: "code", project: "my-project" },
    status: "running",
    context: "Processing data"
  });
  
  const data = await fetchData();
  const result = processData(data);
  
  await orbit_result({
    agent: { id: "my-agent", project: "my-project" },
    title: "Data Processing Result",
    content: JSON.stringify(result, null, 2),
    trust_level: "conclusion"
  });
  
  await orbit_heartbeat({
    agent: { id: "my-agent", project: "my-project" },
    status: "idle",
    summary: "Processed data, published result"
  });
}
```
