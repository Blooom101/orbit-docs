# Persistent Tables

For ongoing structured data (prospect lists, outreach trackers, pipelines, ledgers), use `orbit_table_write` instead of local CSVs. Tables live in the Vault and persist across agent runs.

## orbit_table_write

```javascript
await orbit_table_write({
  agent: { id: "my-agent", project: "my-project" },
  table_ref: "outreach/q3-prospects",
  title: "Q3 Prospects",
  columns: ["Name", "Email", "Status", "Last Contacted"],
  rows: [
    { Name: "Alice", Email: "alice@...", Status: "Qualified", "Last Contacted": "2026-06-28" },
    { Name: "Bob", Email: "bob@...", Status: "Contacted", "Last Contacted": "2026-06-25" }
  ],
  mode: "append"
});
```

## Parameters

### Required

| Field | Type | Description |
|-------|------|-------------|
| `agent.id` | string | Your agent slug |
| `agent.project` | string | Project slug |
| `table_ref` | string | Stable lowercase slug (e.g., `outreach/q3-prospects`) |
| `title` | string | Display name in Vault |
| `rows` | object[] | Array of row objects keyed by column name |

### Optional

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `columns` | string[] | - | Column names (required on first write) |
| `mode` | string | `append` | One of: `append`, `upsert`, `replace` |
| `key_columns` | string[] | - | For `upsert`: columns that identify a row |
| `note_above` | string | - | Markdown rendered above table |
| `note_below` | string | - | Markdown rendered below table |
| `trust_level` | string | `conclusion` | `authoritative` or `conclusion` |

## Modes

### append (default)
Add rows to the end. Previous data stays.

```javascript
await orbit_table_write({
  table_ref: "outreach/q3-prospects",
  title: "Q3 Prospects",
  columns: ["Name", "Email"],
  rows: [
    { Name: "Charlie", Email: "charlie@..." }
  ],
  mode: "append"
});
```

### upsert
Merge by `key_columns`. Rows with matching keys are updated; new keys are appended.

```javascript
await orbit_table_write({
  table_ref: "outreach/q3-prospects",
  title: "Q3 Prospects",
  columns: ["Email", "Name", "Status"],
  rows: [
    { Email: "alice@...", Name: "Alice Johnson", Status: "Qualified" }  // Updates Alice's record
  ],
  mode: "upsert",
  key_columns: ["Email"]
});
```

### replace
Wipe the table and replace with new data. Use sparingly.

```javascript
await orbit_table_write({
  table_ref: "outreach/q3-prospects",
  title: "Q3 Prospects",
  columns: ["Name", "Email"],
  rows: [...],
  mode: "replace"
});
```

## Persistent Identity

Use a stable `table_ref` (like you do with file `ref`) to maintain the same table across runs:

```javascript
// Run 1: Create table
await orbit_table_write({
  table_ref: "outreach/prospects",
  title: "Prospects",
  columns: ["Name", "Email"],
  rows: [
    { Name: "Alice", Email: "alice@..." }
  ]
});

// Run 2: Add more rows to the same table
await orbit_table_write({
  table_ref: "outreach/prospects",  // Same ref!
  title: "Prospects",
  rows: [
    { Name: "Bob", Email: "bob@..." }
  ],
  mode: "append"
});
```

The Vault shows a single table that evolves. Without a stable `table_ref`, you'd create duplicate tables.

## Reading Tables

Retrieve a table you created:

```javascript
const table = await orbit_table_get({
  agent: { id: "my-agent", project: "my-project" },
  table_ref: "outreach/prospects"
});

// table.rows, table.columns, etc.
```

List all tables you own:

```javascript
const tables = await orbit_table_list({
  agent: { id: "my-agent", project: "my-project" }
});

// tables[0].table_ref, tables[0].title, tables[0].row_count
```

## Patterns

### Ongoing Contact Tracking

```javascript
async function outreachAgent() {
  const hb = await orbit_heartbeat({ ... });

  // Get prior contacts
  const table = await orbit_table_get({
    agent: { id: "outreach", project: "my-project" },
    table_ref: "outreach/contacts"
  });

  const priorContacts = table.rows || [];

  // Add new contacts from today
  const newContacts = await findNewProspects();

  const allRows = [
    ...priorContacts,
    ...newContacts.map(c => ({
      Name: c.name,
      Email: c.email,
      Status: "New",
      DateAdded: new Date().toISOString().split("T")[0]
    }))
  ];

  await orbit_table_write({
    agent: { id: "outreach", project: "my-project" },
    table_ref: "outreach/contacts",
    title: "Prospect Contacts",
    columns: ["Name", "Email", "Status", "DateAdded"],
    rows: allRows,
    mode: "replace"  // Rewrite the whole list
  });
}
```

### Status Updates via Upsert

```javascript
async function updateStatus() {
  // Update only specific rows
  const updates = [
    { Email: "alice@...", Status: "Qualified" },
    { Email: "bob@...", Status: "Contacted" }
  ];

  await orbit_table_write({
    agent: { id: "outreach", project: "my-project" },
    table_ref: "outreach/contacts",
    title: "Prospect Contacts",
    columns: ["Email", "Status"],
    rows: updates,
    mode: "upsert",
    key_columns: ["Email"]  // Match on email
  });
  
  // Only these two rows are updated; others stay the same
}
```

### Run Ledger

Track every execution:

```javascript
async function dailyTask() {
  const startTime = new Date();

  try {
    // Do work...

    const duration = (new Date() - startTime) / 1000;

    await orbit_table_write({
      agent: { id: "scheduler", project: "my-project" },
      table_ref: "operations/run-log",
      title: "Run Log",
      columns: ["Timestamp", "Status", "Duration (s)", "Records Processed"],
      rows: [
        {
          Timestamp: startTime.toISOString(),
          Status: "Success",
          "Duration (s)": duration,
          "Records Processed": 150
        }
      ],
      mode: "append"
    });
  } catch (err) {
    await orbit_table_write({
      agent: { id: "scheduler", project: "my-project" },
      table_ref: "operations/run-log",
      title: "Run Log",
      columns: ["Timestamp", "Status", "Error"],
      rows: [
        {
          Timestamp: startTime.toISOString(),
          Status: "Failed",
          Error: err.message
        }
      ],
      mode: "append"
    });
  }
}
```

## Key Differences vs CSV

| CSV | Orbit Table |
|-----|-------------|
| Stored locally | Stored in Vault |
| Deleted on re-run | Persists across runs |
| Not searchable | Searchable in Vault |
| Invisible to other agents | Readable by other agents |
| Version conflicts | Automatic merging (append/upsert) |

## Best Practices

1. **Reuse before create** - Check `heartbeat.your_trackers` before creating a new table
2. **Use stable refs** - `outreach/prospects`, `operations/log`, `sales/pipeline`
3. **Pick the right mode** - `append` for logs, `upsert` for records, `replace` for full rewrites
4. **Set key columns for upsert** - Defines what "matching row" means
5. **Include metadata** - Add markdown `note_above` explaining the table's purpose
