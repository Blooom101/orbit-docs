# orbit_event

Log agent activity so the operator sees what you're doing as it happens. Events are one-line summaries that appear on the dashboard timeline.

## Usage

```javascript
await orbit_event({
  agent: {
    id: "research-agent",
    project: "my-project"
  },
  message: "Analyzed 150 papers, identified 3 trends",
  kind: "log"
});
```

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `agent.id` | string | Yes | Stable agent slug |
| `agent.project` | string | Yes | Project slug |
| `message` | string | Yes | One-line activity summary (max 50,000 chars, but keep it concise) |
| `kind` | string | No | One of: `log` (default), `milestone`, `system` |

## Message Best Practices

Keep messages concise and actionable:

| Good | Bad |
|------|-----|
| "Analyzed 500 rows, found 3 anomalies" | "Processing data" |
| "Updated pricing page copy per operator request" | "Made changes" |
| "Filed Q3 forecast; confidence 0.85" | "Finished task" |
| "API rate limited, retrying in 60s" | "Error occurred" |

## Message Kinds

### log (default)
Routine progress. Example: "Analyzed 150 papers"

### milestone
Meaningful completion. Example: "Shipped homepage redesign"

Use sparingly; reserved for significant progress.

### system
Errors or system events. Example: "API timeout, retrying"

Use when something went wrong or is noteworthy.

## Response

```json
{
  "ok": true,
  "event_id": "evt-789",
  "timestamp": "2026-06-28T14:30:45Z"
}
```

## Patterns

### Log Work in Progress

```javascript
// Long-running task with progress updates
const items = [...];
for (const item of items) {
  await processItem(item);
}

await orbit_event({
  agent: { id: "processor", project: "my-project" },
  message: `Processed ${items.length} items, filed results`
});
```

### System Event on Error

```javascript
try {
  const data = await fetchAPI();
} catch (err) {
  await orbit_event({
    agent: { id: "fetcher", project: "my-project" },
    message: `API error: ${err.message}; retrying...`,
    kind: "system"
  });
  
  // Retry or handle
}
```

### Milestone at Major Completion

```javascript
// End of run
await orbit_event({
  agent: { id: "daily-digest", project: "my-project" },
  message: "Compiled 47 stories, published digest",
  kind: "milestone"
});
```

## Mandate

**IMPORTANT:** Every user message exchange MUST include at least one `orbit_event` call. This makes your work visible. Silent turns are failed runs.

From the CLAUDE.md:
> "EVERY USER EXCHANGE → call `orbit_event` with your activity summary. One event per turn, minimum. Skipping = invisible to operator = failed run."

## When to Call

- After completing a significant task
- After publishing a result
- On errors or noteworthy events
- At the end of every run (summarize the session)

## Frequency

- Minimum: 1 per run
- Maximum: No hard limit, but 1-5 per run is typical
- Burst: If you're logging many events, space them out (don't fire 100 at once)

## Common Patterns

### Research Agent

```javascript
async function researchAgent() {
  const hb = await orbit_heartbeat({
    agent: { id: "research", type: "code", project: "my-project" },
    status: "running",
    context: "Analyzing Q3 market trends"
  });

  // Do research
  const papers = await searchPapers("Q3 trends");
  
  await orbit_event({
    agent: { id: "research", project: "my-project" },
    message: `Found ${papers.length} papers on trends`
  });

  const analysis = analyzePapers(papers);
  
  await orbit_result({
    agent: { id: "research", project: "my-project" },
    title: "Q3 Market Trends",
    content: analysis,
    trust_level: "conclusion"
  });

  await orbit_event({
    agent: { id: "research", project: "my-project" },
    message: "Filed Q3 trends analysis",
    kind: "milestone"
  });

  await orbit_heartbeat({
    agent: { id: "research", project: "my-project" },
    status: "idle",
    summary: "Researched and analyzed Q3 trends"
  });
}
```

### Scheduled Task (Cron)

```javascript
// Runs every 6 hours
async function backgroundTask() {
  try {
    const hb = await orbit_heartbeat({
      agent: { id: "bg-task", type: "scheduled", project: "my-project" },
      status: "running",
      context: "Hourly data sync"
    });

    const count = await syncData();

    await orbit_event({
      agent: { id: "bg-task", project: "my-project" },
      message: `Synced ${count} records`
    });

    await orbit_heartbeat({
      agent: { id: "bg-task", project: "my-project" },
      status: "idle",
      summary: `Synced ${count} records`
    });
  } catch (err) {
    await orbit_event({
      agent: { id: "bg-task", project: "my-project" },
      message: `Sync failed: ${err.message}`,
      kind: "system"
    });

    await orbit_heartbeat({
      agent: { id: "bg-task", project: "my-project" },
      status: "error",
      summary: "Data sync failed"
    });
  }
}
```
