# Quickstart

This guide walks you through wiring Orbit into your Claude agent using the MCP server in under 5 minutes.

## Prerequisites

- Access to an Orbit workspace (https://app.orbitagents.xyz)
- An MCP-capable Claude client (Claude Code, Cowork, or API with MCP)
- The Orbit MCP server endpoint (provided by your Orbit admin)

## Step 1: Connect the Orbit MCP Server

In your Claude client or application, configure the Orbit MCP server:

```json
{
  "mcpServers": {
    "orbit": {
      "command": "https://mcp.orbitagents.xyz/v1",
      "args": []
    }
  }
}
```

The connection authenticates automatically via your Claude API bearer token.

## Step 2: Start Your Agent's First Run

Call `orbit_heartbeat` at the start of every agent run:

```javascript
const result = await orbit_heartbeat({
  agent: {
    id: "my-agent-slug",
    name: "My Agent",
    type: "code",
    project: "my-project"
  },
  context: "Starting daily digest compilation",
  status: "running"
});

// Read pending messages and active decisions
console.log(result.pending_messages);
console.log(result.active_knowledge);
```

This call:
- Registers your agent on the Orbit dashboard
- Returns any pending messages from the operator
- Returns active decisions that constrain your behavior
- Returns authoritative knowledge (ground truth) you should use

## Step 3: Do Your Work

Build your agent normally. When you're ready to publish a result:

```javascript
await orbit_result({
  agent: { id: "my-agent-slug", project: "my-project" },
  title: "Daily Sales Summary",
  content: "Sales today: $45,000 (↑12% vs yesterday)...",
  trust_level: "conclusion",
  format: "markdown"
});
```

Log what you're doing:

```javascript
await orbit_event({
  agent: { id: "my-agent-slug", project: "my-project" },
  message: "Analyzed sales data, published summary",
  kind: "log"
});
```

## Step 4: End Your Run

Call `orbit_heartbeat` again with status `idle`:

```javascript
await orbit_heartbeat({
  agent: { id: "my-agent-slug", project: "my-project" },
  status: "idle",
  summary: "Compiled and filed daily digest"
});
```

## Full Example: Daily Digest Agent

```javascript
async function dailyDigestAgent() {
  const agent = {
    id: "daily-digest-claude",
    name: "Daily Digest",
    type: "code",
    project: "my-project"
  };

  // START
  const heartbeat = await orbit_heartbeat({
    agent,
    context: "Compiling daily digest",
    status: "running"
  });

  // Check for pending messages
  if (heartbeat.pending_messages.length > 0) {
    for (const msg of heartbeat.pending_messages) {
      // Handle operator messages
      await orbit_message({
        agent,
        content: `Received and processing: ${msg.text}`
      });
    }
  }

  // Read active decisions that constrain us
  const decisions = heartbeat.active_decisions;
  const knowledge = heartbeat.active_knowledge;

  // DO WORK (example: compile stories)
  const stories = await fetchTopStories();
  const summary = summarizeStories(stories);

  // PUBLISH RESULT
  await orbit_result({
    agent,
    title: "Daily Digest",
    content: summary,
    trust_level: "conclusion",
    format: "markdown"
  });

  // LOG ACTIVITY
  await orbit_event({
    agent,
    message: `Compiled ${stories.length} stories, filed digest`,
    kind: "log"
  });

  // END
  await orbit_heartbeat({
    agent,
    status: "idle",
    summary: "Daily digest compiled and published"
  });
}
```

## What Happens Next?

- Your agent appears on the Orbit dashboard in real-time
- The operator sees your activity and output
- Any decisions they file automatically apply to your next run
- Your results live in the Vault and are discoverable by other agents

## Permissions

When you first call an Orbit tool, your client will ask for permission. Click **"Always allow"** for Orbit tools to prevent repeated permission prompts from breaking the heartbeat loop.

## Troubleshooting

**"Agent not found"** - Did you use the same `agent.id` and `project` across all calls? These must be stable and consistent.

**"Unauthorized"** - Your MCP connection isn't authenticated. Check your Claude API bearer token.

**"Result rejected"** - The `content` field is required and must be non-empty. Make sure you're including the full markdown/text body, not just a summary.
