# Quickstart: Connect Your First Agent to Orbit

> Get your first AI agent reporting to Orbit in under 5 minutes. No infrastructure setup, no code required for most integrations.

---

## Before you start

You'll need:
- An Orbit account at [app.orbitagents.xyz](https://app.orbitagents.xyz)
- Your personal MCP endpoint URL — find this in **Settings → MCP** inside your Orbit dashboard. It looks like this:

```
https://app.orbitagents.xyz/api/public/mcp/[your-key]
```

Keep this URL private — it's your agent's authentication key.

---

## Option A — Connect Claude (Recommended for most users)

This is the fastest way to get started. Claude connects to Orbit via MCP (Model Context Protocol) — a standard that lets AI tools share context and tools.

### In Claude.ai (Claude Projects)

1. Open [claude.ai](https://claude.ai) and go to **Settings → Integrations**
2. Click **Add MCP Server**
3. Paste your Orbit MCP URL:
   ```
   https://app.orbitagents.xyz/api/public/mcp/[your-key]
   ```
4. Name it "Orbit" and save
5. Start a new conversation — Claude will automatically connect to Orbit at the start of the session

### In Claude Code (Terminal)

Add Orbit to your `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "orbit": {
      "command": "npx",
      "args": ["-y", "@orbit/mcp-server"],
      "env": {
        "ORBIT_MCP_URL": "https://app.orbitagents.xyz/api/public/mcp/[your-key]"
      }
    }
  }
}
```

Or add it directly via the Claude Code CLI:

```bash
claude mcp add orbit https://app.orbitagents.xyz/api/public/mcp/[your-key]
```

## Option B — Connect via n8n

If you're running automated workflows in n8n, add Orbit as an MCP tool in your workflow:

1. In your n8n workflow, add an **MCP Client** node
2. Set the server URL to your Orbit MCP endpoint
3. The node will expose all `orbit_*` tools — use `orbit_heartbeat` at the start of your workflow and `orbit_result` to file outputs

See the [n8n Integration guide](/guides/integrations#n8n) for a full walkthrough with example workflows.

---

## Option C — Connect Codex or Cursor

Both Codex CLI and Cursor support MCP servers. Add your Orbit MCP URL to your tool's MCP configuration the same way as Claude Code above. Your coding agent will automatically check in with Orbit at the start of each session, read your active decisions, and file its outputs to your vault.

---

## What happens after you connect

Once your agent is connected and runs for the first time:

**On your dashboard**, you'll see the agent appear in your **Orbits** list with a "Live" status badge.

**In your Vault**, the agent's first outputs will start appearing — reports, notes, and any knowledge it files.

**In your Briefing**, the agent shows up in your daily morning overview — what it did, what it produced, when it last ran.

**In your org chart**, the agent appears as a node — you can assign it to a department and give it a role title.

---

## Your first test

After connecting, start a Claude session and ask it to:

> "Check in with Orbit, tell me what decisions are active, and file a test result to the vault."

Claude will call `orbit_heartbeat`, receive your active decisions and knowledge, then call `orbit_result` to file a test entry. You'll see both actions appear on your Orbit dashboard in real time.

---

## What the agent does automatically

Once connected, a properly set up agent will:

- **Call `orbit_heartbeat` at the start of every run** — receiving your active decisions and any knowledge you've approved as ground truth
- **Call `orbit_event` throughout the run** — so you see live progress on your dashboard, not just a result at the end
- **Call `orbit_result` when it completes work** — filing the output to your vault with a trust level
- **Call `orbit_heartbeat` again at the end** — closing the run and recording what shipped

All of this happens automatically for agents using the Orbit skill. For custom setups, see the [API Reference](/api/overview).
