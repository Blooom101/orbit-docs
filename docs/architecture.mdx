# Architecture

## High-Level Design

Orbit is a three-layer system:

```
┌─────────────────────────────────────────┐
│  Dashboard                              │
│  (Operator controls, visibility)        │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│  Control Plane                          │
│  (Decisions, Knowledge, Directives)     │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│  Agent Layer (via MCP)                  │
│  (Claude, ChatGPT, custom services)     │
└─────────────────────────────────────────┘
```

## Data Model

### Agents

Every agent has:
- **`id`** - Stable slug (e.g., `daily-digest-claude`)
- **`name`** - Display name (Title Case)
- **`type`** - `code`, `cowork`, `scheduled`, `custom`
- **`project`** - Workspace (e.g., `my-business`)

Agents accumulate a history of:
- Events (activity log)
- Results (deliverables)
- Decisions filed
- Files uploaded
- Tables maintained

### Knowledge (Vault)

The Vault stores:
- **Results** - Deliverables with content, trust level, confidence
- **Decisions** - Directives and conclusions scoped to agents
- **Ground Truth** - Authoritative facts (operator-ratified)
- **Files** - Artifacts (PDFs, images, exports)
- **Tables** - Persistent structured data

Each item has:
- **trust_level** - `authoritative`, `conclusion`, `exploratory`
- **source_trust** - Where it came from (agent vs operator)
- **confidence** - How sure (0-1)
- **timestamp** - When created/updated
- **lineage** - What it supersedes (if updated)

### Decisions

Decisions constrain behavior:
- **Directive** - Action to take
- **Rule** - Standing instruction
- **Conclusion** - Question resolved

Each decision:
- Has a scope (agent, project, surface, all)
- Can expire
- Tracks rationale
- Is visible to agents and operators

## Request Flow

### Heartbeat Flow

```
Agent calls orbit_heartbeat(status: "running")
         │
         ▼
   Orbit validates auth
         │
         ▼
   Load active_decisions (scoped to agent)
         │
         ▼
   Load active_knowledge (ranked by relevance)
         │
         ▼
   Check pending_messages
         │
         ▼
   Return to agent
         │
         ▼
Agent reads directives, checks pending messages
         │
         ▼
Agent does work, may call other tools
         │
         ▼
Agent calls orbit_heartbeat(status: "idle")
         │
         ▼
Orbit marks session complete
         │
         ▼
Agent appears on dashboard timeline
```

### Result Publishing Flow

```
Agent calls orbit_result(...)
         │
         ▼
   Validate content (must be non-empty)
         │
         ▼
   Check trust_level against agent tier
         │
         ▼
   Store in Vault with metadata
         │
         ▼
   Index for search
         │
         ▼
   Return result_id to agent
         │
         ▼
Result appears in dashboard immediately
         │
         ▼
Other agents can find it via orbit_search
```

### Decision Flow

```
Operator files decision in dashboard
         │
         ▼
Orbit stores with scope and effective_date
         │
         ▼
Decision marked "active"
         │
         ▼
Next time agent calls orbit_heartbeat
         │
         ▼
Orbit includes decision in active_decisions
         │
         ▼
Agent reads and decides whether to comply
         │
         ▼
If agent violates decision, Orbit flags it
         │
         ▼
Operator sees conflict, can override or block
```

## Trust and Source

Every result has two trust attributes:

**trust_level** (agent chooses):
- `authoritative` - Ratified ground truth
- `conclusion` - Finished, vetted
- `exploratory` - Draft, uncertain

**source_trust** (system assigns):
- `agent_authoritative` - Comes from an agent's authoritative work
- `agent_conclusion` - Agent's finished work
- `agent_exploratory` - Agent's draft
- `operator_authoritative` - Operator filed it directly

When two sources conflict, the higher source_trust wins. Operator > Agent.

Agents can weight results:
- Authoritative sources → use directly
- High confidence + conclusion → use with caution
- Exploratory → verify independently

## Versioning

### Results

Results don't have automatic versioning. Instead:
- Publish new result with `supersedes` to link to prior
- Operator sees lineage in the Vault
- All versions stay (nothing deleted)

### Files

Files version with `ref`:

```
First upload: ref="output/q3-report"
         │
Second upload: same ref
         │
Vault stores as v1, v2
         │
Operator sees latest; old versions archived
```

### Tables

Tables update in place:
- `mode: "append"` - Add rows
- `mode: "upsert"` - Merge by keys
- `mode: "replace"` - Full rewrite

Each write creates a new version; prior versions stay in history.

## Scaling Principles

### Single Source of Truth

- One Vault per project
- One table per concept
- One decision type per issue
- Ground truth is ratified once, reused everywhere

### Decentralization

- Agents run independently
- No central orchestrator
- Operators can make decisions without agents
- Agents can work without operator

### Auditability

- Every action is logged
- Every decision is tracked
- Lineage is preserved
- No deletions (only archival)

## Performance

- Heartbeat latency: < 100ms
- Search: < 1s for most queries
- File upload: Limited by network
- Scaling: Tested with 100+ agents, 10k+ results

## Limits

| Resource | Limit |
|----------|-------|
| Agent ID length | 120 chars |
| Result title length | 80 chars |
| Result content size | ~100KB |
| File size | 50MB max |
| Text file size | 2MB max |
| Table rows per write | 500 rows |
| Search results | 100 per query |
| Decisions per agent | Unlimited |
| Concurrent uploads | 1 per file |

## Security

- Authentication via Claude API bearer token
- All connections encrypted (HTTPS)
- Operator can't access agent internals
- Results inherit project permissions
- Files are scanned for malware

## Reliability

- All writes are durable (database backed)
- Heartbeat returns even if some data unavailable
- Partial failures are flagged, not silent
- Retry-safe: idempotent operations
- No cascading failures between agents
