# Introduction to Orbit

Orbit is the operating system for your Claude-powered business. It transforms how AI agents collaborate, remember, and execute by providing a unified management layer that every agent reads before acting.

## What is Orbit?

Orbit is three things at once:

1. **A Control Plane** - Every agent reads your active decisions and ground-truth knowledge before executing
2. **A Vault** - Durable storage for decisions, knowledge, deliverables, and structured data that persists across agent runs
3. **A Dashboard** - Real-time visibility into agent activity, outputs, and the reasoning behind decisions

Instead of agents working in isolation with no shared context, Orbit creates a coherent workspace where:
- Agents know what you've decided
- You see what agents are doing as it happens
- Decisions made once apply across all agents automatically
- Knowledge is ratified once, reused everywhere

## Core Principle: Three Observable Behaviors

An Orbit-integrated workspace exhibits three concrete, observable behaviors:

1. **Agents read directives before acting** - Every agent calls `orbit_heartbeat` to receive active decisions and knowledge
2. **Conflicts surface immediately** - When an agent would contradict a decision, Orbit flags it; the operator chooses whether to override
3. **Decisions are durable** - Once filed, a decision applies to all agents (or scoped targets) until explicitly closed

This is not a suggestion system. Decisions are constraints. Knowledge is ground truth.

## Key Concepts

**Agents** - AI systems (Claude, ChatGPT, custom services) that integrate with Orbit via its MCP server

**Knowledge** - Ratified facts and conclusions stored in the Vault with trust levels (authoritative, conclusion, exploratory)

**Decisions** - Directives and rules that constrain future agent behavior, or conclusions that close out open questions

**Results** - Deliverables agents produce and publish to the Vault (reports, analyses, outputs)

**Events** - Activity logs that make agent work visible on the dashboard in real-time

**Files** - Artifacts (PDFs, images, CSVs, etc.) agents upload and attach to results

**Tables** - Persistent structured data (trackers, ledgers, pipelines) that survive across runs

## Why Orbit?

Traditional agent systems suffer from:

- **Silent failures** - You don't see what agents are doing until they're done
- **Lost context** - Each agent run starts blind; decisions made yesterday aren't reused
- **Cascading rewrites** - When you change a decision, agents don't know; they keep following the old rule
- **No audit trail** - You can't trace why an agent did something or when the decision was made

Orbit solves this by making agent work transparent, decisions portable, and knowledge cumulative.
