title: "Is Orbit right for me?"
description: "Three honest pictures of who Orbit fits — and who it doesn't."
---

Orbit earns its place when you run more than one AI agent and need them to stay coordinated. Here's how to tell if that's you.

## You're the solopreneur running 5 AI tools

You've got Claude for strategy, a scheduled agent for outreach, n8n for lead gen, maybe Codex for code. Each works — but you're the human glue holding them together, copy-pasting context and re-explaining the same rules.

**Orbit fits if:** you're tired of being the memory between your tools and want them to share one brain.

## You're the founder building an AI-native team

You're not hiring people for every role — you're deploying agents. Sales, research, content, ops, each handled by a recurring AI agent. You need to see what they're all doing, set company-wide rules, and trust that they respect them.

**Orbit fits if:** you want an org chart of agents, a daily briefing of their work, and decisions that apply across the whole team.

## You're the operator who lives in Claude

Claude is your primary workspace. You run long, complex sessions and it kills you that each new chat forgets everything the last one learned.

**Orbit fits if:** you want Claude to remember across sessions and file its best work somewhere permanent and searchable.

## When Orbit is *not* the right tool

Be honest with yourself:

- **You only use one AI tool, one chat at a time.** If there's nothing to coordinate, Orbit's shared layer is overhead you don't need yet.
- **You want a fully autonomous agent that needs no oversight.** Orbit is a control layer that keeps *you* in the loop — it's the opposite of fire-and-forget.
- **You need an on-prem, air-gapped deployment today.** Self-hosting is available on request, but the default is cloud.

If none of the three pictures above sound like you, Orbit can wait until your stack grows.
