GitHub is where cloud Claude Code does its best work: it branches, commits, and opens pull requests you review. Here's the flow that keeps things safe and legible.

## The core flow

1. **Branch, don't push to main.** Have Claude work on a feature branch.
2. **Make focused commits.** One logical change per commit, clear messages.
3. **Open a PR.** Claude opens a pull request describing what changed and why.
4. **Review and merge.** You stay the approver — read the diff, request changes, merge.

A good standing rule (set it once): *never commit directly to `main`; always branch and open a PR.* Put it in [CLAUDE.md](/claude-education/folders-context) for one repo, or an [Orbit decision](/concepts/decisions) to bind every coding agent you run.

## Telling Claude what you want

Be explicit about the GitHub action:

> "Create a branch `fix/login-timeout`, fix the bug, open a PR against `main`, and summarize the change."

Claude uses the GitHub CLI / API to do exactly that — no manual git choreography from you.

## Keeping context across PRs with Orbit

A PR review can span days and multiple sessions. Without shared memory, each session re-derives the context. With Orbit:

- The reasoning behind the change is **filed to your Vault**, so a later session (or a different agent) picks up why it was done that way.
- An [Orbit decision](/concepts/decisions) like "auth uses JWT" is read by every agent touching the repo, so reviews stay consistent.
- Architectural conclusions reached in review become **Knowledge** the next agent reads before it acts.

## Safe defaults

- Branch + PR always; gate direct pushes to `main`.
- Keep destructive git operations (force-push, history rewrite) behind explicit approval — see [Permissions](/claude-education/permissions).
- Let humans merge. Claude proposes; you dispose.
