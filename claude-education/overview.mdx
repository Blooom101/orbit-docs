This track teaches you to use Claude in totality: the surfaces, the models, folders, permissions, commands, and scheduled automation — every page assuming you have **Orbit connected**, so what you learn compounds instead of resetting.

## Why "with Orbit connected" changes everything

Claude on its own is powerful but forgetful: every chat, every Code session, every scheduled run starts from zero. Orbit is the shared brain underneath. Once it's connected:

- Claude **reads your decisions and knowledge before it acts** (via `orbit_heartbeat`).
- Claude **files what it produces to your Vault** (via `orbit_result`), so the next session can build on it.
- You **see every run** on your dashboard.

So this isn't "Claude tips" in a vacuum. It's how to drive Claude as part of a system that remembers.

## The one mental model to keep

Throughout this track, remember the loop: **Claude reads context from Orbit → does work → files results back to Orbit → you review → it becomes context for next time.** Everything below is just learning to run that loop well across different surfaces.
