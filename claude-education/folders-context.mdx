Claude Code understands your project through its **folder structure** and special **`CLAUDE.md`** memory files. Get these right and Claude stops guessing and starts following your conventions.

## How Claude reads a folder

When you work in a directory, Claude Code explores it on demand — opening the files relevant to your task rather than ingesting everything. The cleaner and more conventional your structure, the better it navigates. Good folder hygiene is good Claude hygiene.

## CLAUDE.md — your project's standing rules

A `CLAUDE.md` file is durable memory Claude reads automatically. Put the things you'd otherwise repeat every session:

```markdown
# Project rules
- Use TypeScript, no `any`.
- Tests live next to the file they test.
- Never edit files in /legacy.
- Run `npm test` before declaring done.
```

### Where it lives

- **Project root `CLAUDE.md`** — rules for this repo, shared with your team (check it in).
- **Global `~/.claude/CLAUDE.md`** — your personal rules across every project.
- **Subfolder `CLAUDE.md`** — rules scoped to one part of a big repo.

Claude reads the relevant ones automatically. No need to paste them into each prompt.

## CLAUDE.md vs Orbit decisions

These work together, at different scopes:

| | `CLAUDE.md` | Orbit Decisions |
|---|---|---|
| Scope | One repo / one machine | Every agent, every tool, everywhere |
| Read by | Claude Code in that folder | All connected agents (Claude, Codex, Cowork…) |
| Best for | Repo conventions, build commands | Company-wide rules, positioning, cross-tool policy |

Rule of thumb: **repo-specific mechanics** go in `CLAUDE.md`; **decisions that should govern your whole AI workforce** go in [Orbit Decisions](/concepts/decisions) so Claude Code, Codex, and your Cowork agents all honor them.

## Practical setup

1. Add a root `CLAUDE.md` with build/test commands and hard "don't touch" rules.
2. Keep a personal `~/.claude/CLAUDE.md` with your style preferences.
3. File anything cross-tool or company-wide as an Orbit decision instead — so it's not trapped in one repo.
