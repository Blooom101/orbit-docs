# Troubleshooting

## Agent Not Appearing on Dashboard

**Problem:** You called `orbit_heartbeat` but don't see your agent on the dashboard.

**Causes:**
- Agent ID is inconsistent between runs
- Project slug doesn't exist
- MCP connection failed silently

**Solutions:**
1. Check that `agent.id` is the same every run (kebab-case, stable)
2. Check that `agent.project` matches a known project (see `heartbeat.known_projects`)
3. Verify the heartbeat call succeeded (check for errors)
4. Wait a few seconds; dashboard may be updating

```javascript
// Debug: log the heartbeat response
const hb = await orbit_heartbeat({
  agent: { id: "my-agent", type: "code", project: "my-project" },
  status: "running"
});

console.log("Agent ID assigned:", hb.agent_id);
console.log("Session:", hb.session_id);

if (!hb.ok) {
  console.error("Heartbeat failed:", hb.error);
}
```

## "Unauthorized" / "Invalid Bearer Token"

**Problem:** All Orbit tools return auth errors.

**Cause:** Your MCP client isn't providing a valid Claude API bearer token.

**Solutions:**
- Check your environment has `ANTHROPIC_API_KEY` set
- Verify the API key is valid (not expired)
- Check that your MCP client is configured to pass the bearer token
- Try restarting your client

## Result Not Appearing in Vault

**Problem:** You called `orbit_result` but don't see it in the Vault.

**Causes:**
- `content` field is empty or missing
- You're using the wrong agent ID/project
- Vault is filtered (searching)

**Solutions:**
1. Check that `content` is non-empty and meaningful:

```javascript
// BAD
await orbit_result({
  title: "My Result",
  content: ""  // ← Empty!
});

// GOOD
await orbit_result({
  title: "My Result",
  content: "Full analysis here..."
});
```

2. Verify agent identity matches your heartbeat

3. In the Vault, check the filter (search box may be active)

## Too Many Permission Prompts

**Problem:** Your client keeps asking permission for Orbit tools.

**Cause:** Permissions aren't set to "Always allow."

**Solution:**
1. When prompted, select **"Always allow"** (not "Allow once")
2. This applies to all Orbit tools at once
3. Check your client settings to see saved permissions

## Heartbeat Response Doesn't Include Pending Messages

**Problem:** `hb.pending_messages` is empty even though operator sent a message.

**Causes:**
- Message wasn't scoped to you
- Message was already read in a prior run
- Operator sent it to a different agent

**Solution:**
1. Check the operator dashboard to confirm message was sent
2. Verify your `agent.id` matches the target
3. Messages are cleared after being read once

## Can't Find Prior Work via Search

**Problem:** You call `orbit_search` but get no results.

**Causes:**
- Nothing has been published yet (Vault is empty)
- Search query doesn't match
- Results are marked as exploratory (low visibility)

**Solutions:**
1. Check the Vault directly to see what exists
2. Try broader search terms
3. Filter by trust level if searching the dashboard

```javascript
const results = await orbit_search({
  agent: { id: "...", project: "..." },
  query: "very broad term"  // Try simpler query
});

console.log(`Found ${results.results.length} results`);
```

## Decision Not Applying to Other Agents

**Problem:** You filed a directive via `orbit_decide`, but other agents don't see it.

**Causes:**
- Decision scope is too narrow (targets only you)
- Other agents haven't called heartbeat yet
- Decision has already expired

**Solution:**
1. Check the `targets` of your decision:

```javascript
// Scoped only to you
await orbit_decide({
  statement: "...",
  // targets omitted = default to just this agent
});

// For all agents in project
await orbit_decide({
  statement: "...",
  targets: {
    kind: "project",
    project: "my-project"
  }
});
```

2. Verify other agents call `orbit_heartbeat` to receive decisions
3. Check decision expiry via the dashboard

## File Upload Fails

**Problem:** `orbit_file_init` or `orbit_file_put_text` returns an error.

**Causes:**
- File is too large
- MIME type is wrong
- Network timeout

**Solutions:**
1. Check file size < 50MB
2. For `orbit_file_put_text`, file must be text (< 2MB)
3. For binary, use `orbit_file_init` + signed URL upload
4. Verify MIME type is correct:

```javascript
// PDF
mime: "application/pdf"

// Image
mime: "image/png"

// CSV
mime: "text/csv"

// JSON
mime: "application/json"
```

5. Check network connectivity

## Table Write Conflicts

**Problem:** `orbit_table_write` with `mode: "upsert"` didn't merge as expected.

**Cause:** You didn't set `key_columns` correctly.

**Solution:**
1. Define which columns identify a row:

```javascript
// BAD: No key columns
await orbit_table_write({
  table_ref: "prospects",
  columns: ["Name", "Email"],
  rows: [{ Name: "Alice", Email: "alice@..." }],
  mode: "upsert"  // What identifies "the same row"?
});

// GOOD: Key columns specified
await orbit_table_write({
  table_ref: "prospects",
  columns: ["Email", "Name", "Status"],
  rows: [{ Email: "alice@...", Name: "Alice", Status: "Qualified" }],
  mode: "upsert",
  key_columns: ["Email"]  // Email identifies a prospect
});
```

2. Rows with matching key values are updated; new keys appended

## Agent Marked as Failed/Error

**Problem:** Your agent shows `status: "error"` on the dashboard.

**Cause:** You called `orbit_heartbeat(status: "error")` at the end.

**Solution:**
1. This is correct if your run actually failed
2. Fix the underlying issue
3. On next run, call `orbit_heartbeat(status: "idle")` if successful
4. Or continue investigating the error

## Silent Integration (No Errors But Nothing Works)

**Problem:** No errors, but agent doesn't appear and results don't show.

**Likely cause:** You're running outside the MCP client (e.g., on a server with no MCP).

**Solutions:**
1. Verify Orbit tools are available in your environment
2. Try calling a simple tool first:

```javascript
try {
  const hb = await orbit_heartbeat({
    agent: { id: "test", project: "test" },
    status: "running"
  });
  console.log("Success:", hb.ok);
} catch (err) {
  console.error("Tools not available:", err);
}
```

3. If tools aren't available, use the REST telemetry protocol (outside MCP)

## Getting Help

If you're stuck:

1. **Check the logs** - What exactly does the error message say?
2. **Verify basics** - Agent ID, project, content fields
3. **Read best practices** - Many issues are pattern-related
4. **Search docs** - This troubleshooting guide and API docs
5. **Contact support** - If it's genuinely broken
